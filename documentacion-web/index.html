Inventario Emocional App

Sistema de gestión emocional para miembros de Al-Anon

Python Kivy MongoDB Chat en Tiempo Real

Descripción General del Proyecto

Inventario Emocional es una aplicación móvil diseñada específicamente para los miembros de Al-Anon, una organización que brinda apoyo a familiares y amigos de personas con problemas de alcoholismo.

Objetivo Principal

La aplicación permite a los usuarios registrar y monitorear sus estados emocionales diarios de manera estructurada, facilitando el proceso de autoconocimiento y recuperación emocional que es fundamental en el programa de Al-Anon.

Usuarios del Sistema

Miembros

Usuarios principales que registran su inventario emocional, consultan estadísticas y participan en el chat grupal.

Administradores

Gestión de usuarios, moderación del chat y supervisión general del sistema.

Valor Agregado

  • Registro estructurado de emociones diarias
  • Visualización de patrones emocionales mediante estadísticas
  • Seguimiento histórico de inventarios
  • Comunidad de apoyo mediante chat en tiempo real
  • Privacidad y seguridad de los datos personales

Requisitos y Especificaciones

Requisitos Funcionales

RF-01: Autenticación

El sistema debe permitir el login diferenciado para miembros y administradores.

RF-02: Registro de Inventario

Los miembros deben poder registrar su inventario emocional diario con campos estructurados.

RF-03: Consulta de Historial

El sistema debe permitir consultar inventarios previos ordenados por fecha.

RF-04: Generación de Estadísticas

El sistema debe generar gráficos y estadísticas de los datos emocionales registrados.

RF-05: Chat en Tiempo Real

Debe existir un chat general para la comunicación entre miembros.

RF-06: Gestión Administrativa

Los administradores deben poder gestionar usuarios y moderar contenido.

Requisitos No Funcionales

RNF-01: Usabilidad

Interfaz intuitiva y fácil de usar para usuarios de todas las edades.

RNF-02: Rendimiento

Respuesta del sistema en menos de 2 segundos para operaciones comunes.

RNF-03: Seguridad

Encriptación de contraseñas y protección de datos sensibles.

RNF-04: Disponibilidad

Sistema disponible 24/7 con conexión a internet.

RNF-05: Escalabilidad

Capacidad de soportar crecimiento en número de usuarios.

RNF-06: Compatibilidad

Funcional en dispositivos Android e iOS.

Estadística

Modelado - Diseño (Fase de Diseño del Sistema)

Esta fase del modelado se enfoca en el diseño técnico de la solución. Aquí se especifica CÓMO el sistema implementará los requisitos identificados en la fase de análisis, definiendo la arquitectura, las clases de la solución y las interacciones detalladas entre componentes.

1. Diagrama de Clases de la Solución

El diagrama de clases de la solución muestra la arquitectura técnica del sistema utilizando el patrón Model-View-Controller (MVC). Incluye paquetes para reflejar la arquitectura del sistema, con separación clara entre capas de presentación (View), lógica de negocio (Controller) y acceso a datos (Model).

Arquitectura MVC - Paquetes del Sistema

View
Capa de Presentación (View)

9 pantallas principales:

  • WelcomeScreen - Pantalla de bienvenida
  • LoginMiembroScreen - Login de miembros
  • LoginAdminScreen - Login de administradores
  • DashboardMiembroScreen - Dashboard principal
  • FormularioInventarioScreen - Formulario de registro
  • HistorialMiembroScreen - Consulta de historial
  • EstadisticasScreen - Visualización de gráficos
  • ChatGeneralScreen - Chat en tiempo real
  • DashboardAdminScreen - Panel administrativo
Controller
Capa de Lógica de Negocio (Controller)

6 controladores principales:

  • UsuarioController - Gestión de usuarios y autenticación
  • InventarioController - CRUD de inventarios emocionales
  • EstadisticasController - Cálculo y generación de estadísticas
  • ChatController - Manejo de mensajes en tiempo real
  • AdminController - Funciones administrativas
  • SesionController - Gestión de sesiones de usuario
Model
Capa de Acceso a Datos (Model)

4 modelos de datos:

  • DatabaseConnection - Conexión a MongoDB
  • UsuarioModel - Operaciones CRUD de usuarios
  • InventarioModel - Operaciones CRUD de inventarios
  • MensajeModel - Operaciones CRUD de mensajes
